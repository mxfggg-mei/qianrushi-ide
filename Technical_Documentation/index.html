核心功能

自动模式

基于光照和温度传感器数据，自动控制照明和风扇设备，实现智能化环境调节。

手动模式

支持按键手动控制，提供4种状态切换，满足用户自定义控制需求。

实时监测

OLED显示屏实时显示系统状态，包括模式、光强、温度、设备状态和WiFi连接状态。

物联网接入

支持WiFi连接和MQTT协议，实现远程控制和状态上报，支持阈值配置和设备管理。

模块功能详解

EXTI

外部中断与按键处理

处理按键输入，支持短按和长按识别。KEY1作为系统使能开关，KEY2支持短按切换状态和长按2秒切换模式。

exti_init() exti_task() key_init()
RELAY

继电器控制

管理两个继电器输出，支持自动和手动两种模式。自动模式下根据传感器数据控制照明和风扇；手动模式下通过按键切换4种状态组合。

relay_init() relay_task() relay_auto_run() relay_manual_run()
SENSOR

传感器驱动

光照传感器使用ADC采集模拟信号，温度传感器使用DS18B20通过1-Wire协议通信。支持原始数据读取和等级判断。

light_sensor_read_raw() light_sensor_get_level() temp_sensor_read_celsius() temp_sensor_get_level()
DISPLAY

OLED显示

使用U8g2库驱动SH1106 OLED显示屏，实时显示系统模式、传感器数据、设备状态和WiFi连接状态。

display_init() display_update_info()
RGB

WS2812 RGB控制

使用Adafruit_NeoPixel库控制WS2812 RGB灯，实现非阻塞渐变动画。根据系统状态显示不同的颜色模式。

ws2812_init() ws2812_boot_sequence() ws2812_task() fade_to()
WiFi

WiFi连接管理

自动连接保存的WiFi网络，支持串口配置SSID和密码。提供连接状态监测和自动重连功能。

wifi_init() wifi_task() wifi_serial_config() wifi_save_config()
MQTT

MQTT通信

基于PubSubClient库实现MQTT协议，支持状态上报、远程控制、阈值配置和设备重启命令。

mqtt_init() mqtt_task() mqtt_report_status() mqtt_callback()

操作指南

详细的系统操作说明

自动模式

系统上电默认模式

工作原理

系统根据光照传感器和温度传感器的数据，自动控制照明和风扇设备：

  • 当光强低于设定阈值时，自动开启照明灯(RELAY1)
  • 当温度高于设定阈值时，自动开启风扇(RELAY2)
  • 阈值可通过串口或MQTT配置，配置保存在Flash中
温度阈值 默认 28°C
光照阈值 默认 119 lux

手动模式

长按KEY2 2秒切换

工作原理

用户通过按键手动控制设备状态，短按KEY2循环切换4种状态：

状态0 灯: OFF 风扇: OFF
状态1 灯: OFF 风扇: ON
状态2 灯: ON 风扇: OFF
状态3 灯: ON 风扇: ON

按键操作

KEY1 GPIO20

系统使能

自锁按键，按下使能系统，释放关闭系统。系统关闭时所有设备停止工作。

KEY2 GPIO21

模式切换 / 状态切换

  • 短按 (2ms-1s): 手动模式下切换设备状态
  • 长按 (≥2s): 在自动/手动模式之间切换

RGB指示灯状态

状态 颜色模式 说明
系统正常 红→绿→蓝→灭 WiFi和MQTT均已连接
WiFi断开 红→绿→灭 WiFi未连接，正在重连
MQTT断开 红→绿→灭→蓝→灭→红 WiFi已连接，MQTT未连接
仅灯亮 红→灭→绿→灭→蓝→灭 RELAY1开启
仅风扇 红→绿→蓝→灭 RELAY2开启
灯和风扇 红→灭→红→灭→绿→灭→绿→灭→蓝→灭→蓝→灭 RELAY1和RELAY2均开启

主题规范

状态上报主题

chemctrl/{device_id}/status

设备主动上报状态信息，包括温度、光照、模式、继电器状态等。

命令接收主题

chemctrl/{device_id}/command

接收远程控制命令，支持设置继电器、切换模式、配置阈值等操作。

消息格式

状态上报

PUBLISH 
{
  "temperature": 25.5,
  "light": 150,
  "mode": "auto",
  "key1_lock": true,
  "relay3": true,
  "relay4": false,
  "temp_threshold": 28.0,
  "light_threshold": 119
}

设置继电器

SUBSCRIBE 
{
  "cmd": "set_relay",
  "relay": 3,
  "value": true
}

relay: 3=照明, 4=风扇; value: true=开, false=关

切换模式

SUBSCRIBE 
{
  "cmd": "set_mode",
  "mode": "auto"
}

mode: "auto"=自动模式, "manual"=手动模式

获取状态

SUBSCRIBE 
{
  "cmd": "get_status"
}

触发设备立即上报状态，5秒内限制一次

设置阈值

SUBSCRIBE 
{
  "cmd": "set_threshold",
  "temp": 28.0,
  "light": 119
}

可单独设置温度或光照阈值，配置保存到Flash

设备重启

SUBSCRIBE 
{
  "cmd": "reboot"
}

2秒后设备重启

命令 说明

配置流程

1

打开串口终端

使用Arduino IDE串口监视器或其他串口工具，波特率设置为115200。

2

发送配置命令

输入 mqtt_config 进入配置模式。

3

输入参数

按照提示依次输入服务器地址、端口、用户名、密码和设备ID。按回车保持默认值，输入esc退出。

4

验证配置

输入 mqtt_status 查看配置是否生效。

故障排查

常见问题及解决方法

设备无响应

可能原因

  • KEY1未按下，系统未使能
  • 电源电压不足
  • 接线错误

解决方法

  • 确保KEY1按下使能系统
  • 检查电源输入（建议5V/2A）
  • 对照引脚分配表检查接线

WiFi连接失败

可能原因

  • WiFi信号弱
  • SSID或密码错误
  • 网络加密方式不支持

解决方法

  • 靠近路由器或使用信号更强的网络
  • 使用串口重新配置WiFi
  • 确保网络使用WPA/WPA2加密

MQTT连接失败

可能原因

  • 服务器地址或端口错误
  • 用户名或密码错误
  • 网络不通

解决方法

  • 检查MQTT服务器配置
  • 确认网络连接正常
  • 使用 mqtt_status 查看状态

传感器数据异常

可能原因

  • 传感器接线错误
  • 传感器损坏
  • ADC配置错误

解决方法

  • 检查传感器接线
  • 更换传感器测试
  • 检查ADC衰减配置

继电器不工作

可能原因

  • 继电器模块未供电
  • 控制引脚接线错误
  • 继电器损坏

解决方法

  • 检查继电器模块电源（通常需要5V）
  • 对照引脚分配表检查接线
  • 更换继电器模块测试

OLED显示异常

可能原因

  • I2C接线错误
  • 显示屏损坏
  • I2C地址冲突

解决方法

  • 检查SDA(GPIO4)和SCL(GPIO5)接线
  • 尝试更换OLED显示屏
  • 确保没有其他I2C设备占用同一总线

调试技巧

🔍

查看串口输出

打开串口监视器（波特率115200），系统会输出详细的运行日志，包括传感器数据、设备状态和错误信息。

📊

观察RGB指示灯

WS2812 RGB灯显示系统状态，不同的颜色模式对应不同的运行状态，可快速判断系统是否正常。

🌐

MQTT消息测试

使用MQTT客户端工具（如MQTTX）订阅设备状态主题，发送控制命令测试设备响应。

🔄

恢复默认配置

如果配置出现问题，可使用 mqtt_clear 清除MQTT配置，系统将恢复默认值。
